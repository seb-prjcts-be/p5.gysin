API reference

System

p5.gysin is plain script files: one core, and two optional voices for permutation and type. Your own sketch.js always comes last.

Start here

p5.gysin has three library files

  • p5.js is the creative-coding foundation. Load it when you want GysinPlot to draw the generated paths on a canvas.
  • p5.gysin.min.js is the core library. It provides GysinPlot, drawing methods such as textCutup(), and SVG, JSON, and HPGL export.
  • p5.gysin.text.min.js is the permutation voice. It provides GysinText.permute(). Load it whenever you want to generate permutation poems.
  • p5.gysin.underwood.min.js is the typed voice. It provides plot.underwood(), the period-correct typewriter.

Load what you need, in this order - with p5.js first when you draw to a canvas:

<script src="https://cdn.jsdelivr.net/gh/seb-prjcts-be/p5.gysin@v0.4.0/p5.gysin.min.js"></script> <script src="https://cdn.jsdelivr.net/gh/seb-prjcts-be/p5.gysin@v0.4.0/p5.gysin.text.min.js"></script> <script src="https://cdn.jsdelivr.net/gh/seb-prjcts-be/p5.gysin@v0.4.0/p5.gysin.underwood.min.js"></script>

The voices are optional. Both are dependency-free; p5.gysin.text.min.js even runs without p5.js or GysinPlot.

Three quickstarts, none longer than twenty-eight lines: rub, chant, plotter. Each page shows its entire sketch.

Basic Use

let plot; function setup() { createCanvas(800, 800); plot = new GysinPlot({ seed: 1960 }); plot.textCutup("RUB OUT THE WORD", 80, 180, { size: 72, breathe: 2, dropout: 0.12, repeat: 2, rubout: 0.25 }); plot.line(80, 260, 720, 280, { breathe: 1, dropout: 0.05, overshoot: 8 }); plot.draw(); }

Constructor and Instance Mode

new GysinPlot(options) accepts seed, width, height, page, and default human, style, and export objects. In p5 instance mode, create the plot after createCanvas():

const plot = p.createGysinPlot({ seed: 1960 });

The helper links drawing and automatic export dimensions to that p5 instance. Capture methods return a shape id. Lookup and mutation methods return detached snapshots, so use update() instead of modifying data returned by get().

Shape Methods

  • plot.line(x1, y1, x2, y2, options)
  • plot.rect(x, y, w, h, options)
  • plot.circle(x, y, diameter, options)
  • plot.polygon(points, options)
  • plot.path(points, options)
  • plot.text(value, x, y, options)
  • plot.textCutup(value, x, y, options)

Fields and Marks

Four capture methods fill a region instead of drawing one shape. Each returns ordinary shape ids (or cell rectangles), so everything they write stays addressable.

const cells = plot.grid(40, 40, 480, 320, 3, 2); // frame grid; returns cell rects plot.letters("RUB OUT", 60, 60, 200, 120); // a decaying letter field plot.symbols(280, 60, 200, 120, { set: "+#&" }); // typewriter operator texture plot.asemic(60, 220, 160, 120); // a cursive, wordless tangle
  • plot.grid(x, y, w, h, cols, rows, options) draws an ink-bled frame grid and returns its cell rectangles ({ x, y, w, h, col, row }) so a composition can place fields declaratively. Options: gap, outer: false (skip the outer frame), cells: false (return rectangles without drawing them), and frame - an options object passed to every drawn rect, including angle/pivot.
  • plot.letters(source, x, y, w, h, options) fills the box with rows of letters drawn from the source phrase, clustered by repetition so words dissolve into a letter field. Returns one id per row.
  • plot.symbols(x, y, w, h, options) fills the box with typewriter operator glyphs. set (default +"#&/) chooses the glyphs; cluster: true repeats them in runs.
  • plot.asemic(x, y, w, h, options) draws a compact, handwriting-like tangle - writing with no words. loops and detail set the density; every other path() option passes through.
  • All four are auto-seeded by call order, so redrawing the same composition reproduces the same fields; pass seed to pin one. Both field methods accept size, lineHeight, letterSpacing, and every trace option.

Shape, Text, Style, and Export Options

  • path: set closed: true for an exact closed contour.
  • text: size, font, sampleFactor, lineHeight, and letterSpacing.
  • textCutup: all text options plus slices, sliceOffset, and sliceDropout.
  • Style: stroke, positive strokeWeight, and clamped alpha.
  • Export per shape: simplify, minSegmentLength, and layer.

Invalid geometry, duplicate ids, excessive sampling, more than 1,000 repeats, and route optimization above 2,000 traces raise an error before export.

Text

When the supplied p5 font exposes its outline path, p5.gysin keeps separate glyph contours and counters. Otherwise it uses font.textToPoints(); without a font, it falls back to a simple vector alphabet.

For p5.js 2.x, load fonts with async setup() and await loadFont(...) before passing the font to plot.text().

Trace Parameters

  • density: positive point sampling density; default 1.
  • breathe: the hand tremor that keeps a line alive - maximum point displacement in pixels; default 0.
  • dropout: clamped 0–1 probability that a point interrupts a path.
  • hesitate: clamped 0–1 probability of small mechanical hesitations.
  • repeat: whole number from 1 through 1000; default 1.
  • drift: non-negative global offset between repeats.
  • bleed: clamped 0–1 share of written contour length selected for additive ink accumulation.
  • bleedPasses: maximum extra passages per selected fragment, from 1 through 3; default 2.
  • bleedSpread: coherent offset between bleed passages; at least 0.1 when bleed is active.
  • bleedCluster: positive target length for contiguous selected fragments; default 18.
  • rubout: clamped 0–1 strength of generated erase zones.
  • fray: non-negative amount of loose edge strokes.
  • pressure: clamped 0–1 screen weight and alpha variation.
  • overshoot: non-negative extension of open path endpoints.
  • segmentLength: positive target spacing; default 8.
  • glyphJitter: non-negative per-glyph variation (rotation, shift, scale) so no two rendered letters are identical; default 0.35, 0 is mechanically exact.
  • fill: "hatch" (parallel strokes) or "cross" (two perpendicular hatchings) fills closed contours; even-odd, so letter counters stay open. Fill strokes carry role: "fill", stats() reports fillPaths and fillLength, and tool: "blade" leaves fill out of the export. House use: letterforms via a real font - not empty tone planes.
  • hatchSpacing: distance between fill strokes; default 2, minimum 0.25.
  • hatchAngle: rotation of the hatching in degrees; default 0.
  • seed: optional deterministic seed for one shape.

Ink accumulation

dropout and rubout remove material. bleed is their additive counterpart: it selects seeded, contiguous trace fragments and adds a bounded number of coherently offset passages. It does not repeat the complete glyph or shape.

plot.text("RUB OUT THE WORD", 80, 180, { bleed: 0.22, bleedPasses: 2, bleedSpread: 0.8, bleedCluster: 18 });

Generated bleed traces carry data-role="bleed" and data-pass in SVG. Statistics expose bleedPaths, bleedLength, overdrawRatio, and maxLocalPasses.

Intent Verbs

The methods above are primitives that you compose yourself. An intent verb is one call that composes several primitives into a finished gesture, with strong defaults, so the common case is a single line. The primitives stay underneath as the escape hatch.

plot.rub(text, x, y, options) draws a word worn away in three copies - legible, cut up, then an asemic scribble that buries the last one. It composes text, textCutup, and asemic.

plot.rub("RUB OUT", 46, 248); // the whole gesture, defaults plot.rub("RUB OUT", 46, 248, { decay: 2 }); // one knob scales the wear plot.rub("RUB OUT", 46, 248, { font: outlineFont }); // fills the legible head
  • decay: scales breathe/drift/dropout/rubout/fray across every copy at once; default 1, 0 draws clean.
  • size (default 46), font (fills the legible head), stroke.
  • tail: the asemic burial of the last copy; default true, set false to leave the word unburied.
  • stepX/stepY: the drift between copies. stages/tangles replace the built-in recipe wholesale.

rub() returns the id of every copy and tangle, so each can be frozen, rerolled, updated, or exported like any other shape. The worn_word example is this one call.

The Turned Sheet

Every shape takes angle (degrees) and pivot. This is the physical gesture of the studio - write, turn the paper, write again - as shape geometry, not as an export afterthought. The turn runs before breathe, dropout, bleed and hatching, so every disturbance lives on the turned line, and update() can turn any standing shape.

plot.text("RUB OUT", 80, 120, { angle: 90 }); // about its pen-down anchor plot.text("RUB OUT", 80, 120, { angle: 45, pivot: "center" }); // about its own middle plot.rect(60, 60, 200, 90, { angle: 30, pivot: { x: 60, y: 60 } }); // about an explicit pin plot.update(id, { angle: 180 });
  • pivot: "anchor" (default): text turns about its pen-down point, rect and circle about their centre, path and polygon about their centroid, line about its midpoint.
  • pivot: "center" turns about the sampled bounds; an { x, y } object sets the pin explicitly.
  • angle: 0 stays byte-identical to a shape without the option. Page-level page.rotation is unchanged and still means the whole exported sheet.

plot.lattice(text, x, y, w, h, options) composes the turn into the finished method: a phrase written in rows that fill a field, then the sheet turned and written across again. Each row cycles the word order by one, so the permutation runs inside the turning; later passes wear more.

plot.lattice("RUB OUT THE WORD", 60, 60, 480, 480); // two passes, 0 and 90 plot.lattice("JUNK IS NO GOOD BABY", 60, 60, 480, 480, { turns: [0, 90, 180], size: 16, wear: 1.6 });
  • turns (default [0, 90]): the sheet's angles, one written pass per entry. Near-perpendicular passes write along the field's other axis so their rows still fill it.
  • wear (default 1): one knob that scales breathe/dropout/drift per pass; 0 writes every pass clean.
  • size (default 14), lineHeight, letterSpacing, stroke, font, alpha, layer (default "lattice"); extra passes any other trace option to every row.
  • Returns the id of every written row, pass by pass - freeze the first pass and only the later hand rerolls. The rotations example is this sheet.

Optional Text Permutations

p5.gysin.text.min.js returns ordinary strings that can be used on their own or drawn through the core library.

<script src="https://cdn.jsdelivr.net/gh/seb-prjcts-be/p5.gysin@v0.4.0/p5.gysin.text.min.js"></script> const lines = GysinText.permute("I LOVE YOU", { seed: 1960, limit: 6, order: "walk" }); lines.forEach((line, index) => { plot.textCutup(line, 40, 65 + index * 55, { size: 28, slices: 12 + index, sliceOffset: 1 + index }); });

The first line stays almost completely readable. Each following line increases the slice offset by one pixel, so the poem gradually moves from legible text toward a stronger cut-up.

Orders: walk, random, lexical, and rotate. The source line stays first, duplicate words produce only unique lines, and limit is capped at 1,000.

plot.chant(text, x, y, options) is the intent verb for that whole recipe: one call takes a phrase through its permutations and sends every new order through textCutup(), each line cut a little deeper. It appears on every plot once this addon is loaded.

plot.chant("CUT ARRANGE GLUE", 60, 90); // the whole poem, defaults plot.chant("CUT ARRANGE GLUE", 60, 90, { descent: 0 }); // every line cut equally plot.chant("CUT ARRANGE GLUE", 60, 90, { order: "rotate", breathe: 1 });
  • lines (default 5): how many permuted lines return; order and seed pass to permute().
  • descent (default 1): scales how much deeper each line is cut; 0 cuts them all equally light.
  • size, leading, slices, sliceOffset set the type; every other textCutup() option (breathe, dropout, bleed, layer, ...) passes straight through.

Optional Typewriter (underwood)

p5.gysin.underwood.min.js adds one verb: plot.underwood(text, x, y, options) types period-correct single-stroke letters at a fixed pitch - ten characters per inch, six lines per inch, light strike wear. The face carries both voices, capitals and lowercase, plus marks like + # & that the built-in vector alphabet lacks.

plot.underwood("KICK THAT HABIT MAN", 60, 90); plot.underwood("HEADING", 60, 130, { underline: 1 }); plot.underwood("BOLD?", 60, 160, { bold: true }); // struck twice plot.underwood("the small voice.", 60, 190);

The machine has no bold and no italic: bold: true strikes the same key twice, underline draws the underscore ladder, and wear (0 and up) is the one ageing knob. Every letter is a single pen line, so it plots exactly as typed.

That grammar of emphasis has a name: Sperrsatz. One face, at one size, upright - so a word was made to matter by spacing it out, striking it twice, or ruling a line beneath it. The options above are not features added to a font; they are the whole typographic voice the machine ever had, kept on purpose.

Addressing

const titleId = plot.text("RUB OUT", 80, 180, { breathe: 2 }); plot.freeze(titleId); plot.update(titleId, { stroke: "#b5362b" }); plot.thaw(titleId); plot.regenerate(titleId); plot.reroll(titleId); // explicit new seeded variation plot.remove(titleId); plot.select(titleId); // id-less calls now target this shape plot.freeze(); // same as plot.freeze(titleId) plot.setSeed(1961); // re-seed and regenerate every unpinned shape plot.clear(); // empty the sheet: shapes, ids, and selection

regenerate() recalculates with the same seed. Use reroll() only when you want a new deterministic variation. select() sets the default target for id-less addressing calls; setSeed() changes the plot seed and regenerates every shape without an explicit seed of its own; clear() removes all shapes and the selection.

Export

const page = { width: 210, height: 297, units: "mm", margin: 10, scale: 0.25, clip: true }; const svg = plot.exportSVG({ page, optimize: true, tool: "pen" }); const json = plot.exportJSON(); const hpgl = plot.exportHPGL({ page, penMap: { frame: 1, type: 2 }, speed: 20 }); const stats = plot.stats({ page, tool: "pen", drawSpeed: 20, travelSpeed: 60 }); plot.downloadSVG("gysin.svg", { page, optimize: true }); plot.downloadJSON("gysin.json"); plot.downloadHPGL("gysin.hpgl", { page });

The page model applies units, margins, origin, rotation, scale, and optional clipping to SVG, HPGL, and statistics. Route optimization is opt-in so the original drawing order remains the default.

tool: "pen" preserves bleed and explicit repeats. tool: "blade" automatically removes every second and later passage, including legacy repeat traces, so a cutter receives only the first physical path.

Supported units are px, mm, cm, and in. Origins are top-left and bottom-left. SVG precision accepts decimals from 0 through 12. HPGL additionally accepts scale, penMap, speed, offsets, and maxPointsPerCommand.

Compatibility

Version 0.4.0 targets p5.js 2.x in global and instance mode. The vector and export core is framework-independent, but p5.js 1.x is not part of the tested support contract.

Input safety

Geometry and sampling values must be finite. Invalid dimensions, duplicate shape ids, generated output above the point budget, and paths that would produce more than 100,000 sampled points raise a clear error; probability values are clamped to the 0–1 range. Active bleed rejects a spread below 0.1 to prevent exact duplicate fragments.
