Font Outlines

Load a real font and p5.gysin fills the letter bodies with hatching while the inner counters stay empty. Both sides show the same glyphs, O8, so the fill mode is the only thing that changes: single-pass hatch on the left, woven cross-hatch twisted 23° off-axis on the right. Below, COUNTER MEMORY is cut up over a hand-inked rule, then decays past it into a field of asemic marks - the memory pulled apart into illegible signs instead of spelled out again. One Weave spacing slider drives all three: tighter weave shuts the bodies, slices the word harder, and tangles the marks.

What to try:

  • Toggle Fill to watch the bodies weave shut.
  • Drag Weave spacing and watch the fill-line count and the cut-up react in the status line.
  • Hit New variation - the seed in the status changes with the drawing.
  • Switch Oswald / Anton for different counters.
  • Download SVG or Download HPGL to send the pen-line total to a real plotter.
Core snippet
// The whole library, in three lines - this alone is a complete sketch:
const plot = new GysinPlot({ seed: 8088 });
const font = await loadFont(FONT_URL);        // Oswald or Anton
plot.text("O8", 64, 190, { font });   // an outline font -> real letter bodies
plot.draw();

// A real outline font is the ONLY extra this example needs. Everything below is
// OPTIONAL disturbance layered on top; defaults are all zero. This example adds:

// 1) the same glyphs filled two ways - single hatch vs woven cross-hatch -
//    so the fill mode is the only variable; counters stay hollow (even-odd rule)
plot.text("O8", 64, 190, { font, fill: "hatch", hatchAngle: 22 });
plot.text("O8", 384, 190, { font, fill: "cross", hatchAngle: 22 + 23 });

// 2) the phrase cut up over a hand-inked rule, tighter weave slicing it harder
plot.textCutup("COUNTER MEMORY", 72, 404, { font, slices: 4, drift: 1.2 });
plot.line(72, 428, 588, 428, { repeat: 2, overshoot: 8 });

// 3) past the rule the words scatter into a field of asemic marks
for (let i = 0; i < 8; i++) plot.asemic(72 + i * 64, 478, 64, 66, { loops: 3 });

plot.draw();

The red arrow points at a counter - the hole inside the O that the hatching leaves empty. Tighter weave = more fill lines, a harder cut-up, and a more tangled asemic field. The status line names the weave angle set on each side.
