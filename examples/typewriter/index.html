underwood()

Typewriter

William Burroughs cut up the world on an Underwood No. 5, the flat black office standard that fixed the look of a typed page for half a century. Its keys carried one face, at one size, upright - a typebar can only strike the single shape cast on it. So a typed sheet has a voice of its own: fixed pitch, six lines to the inch, and the small wear of every strike.

underwood() types in that voice. The defaults alone give a period-correct page - bundled single-stroke letterforms, 10 chars-per-inch / 6 lines-per-inch (the classic Pica machine), light strike wear - so plot.underwood("KICK THAT HABIT MAN", 60, 90) is a complete gesture. Every option is opt-in.

Core snippet
// Load the underwood module, then the whole sketch is one verb:
const plot = new GysinPlot({ seed: 1961 });
plot.underwood("KICK THAT HABIT MAN", 60, 90);   // period typewriter, defaults
plot.draw();

// Period-correct decoration, all opt-in. No bold or italic ever existed
// on a typewriter, so "bold" is just the same key struck twice:
plot.underwood("HEADING", 60, 90, { size: 24, underline: 1 });
plot.underwood("DOUBLE-STRUCK", 60, 130, { bold: true });
plot.underwood("* * * * * * * *", 60, 170);   // a divider it could type
plot.underwood("the small voice.", 60, 210);  // both cases, one face

// wear scales the whole strike irregularity from one knob (0 = clean):
plot.underwood("KICK THAT HABIT MAN", 60, 210, { wear: 0 });

Sperrsatz

One face, one size - yet typed pages still shouted, whispered and ruled themselves into sections. Emphasis was mechanical, not typographic: a word was made to matter by striking it twice (the nearest thing to bold), by underlining it with a run of underscores, by setting it in CAPITALS, or by S p e r r s a t z - spacing the letters apart so the word opened out along the line. Rows of asterisks, hyphens and underscores divided the page; tone came from overprinting characters, never from heavier ink. What the machine could not do - a second weight, a slant, another size, a curve - simply did not happen on the page.

underwood() stays inside those limits on purpose: bold: true is the double strike, underline: 1 the underscore rule, and everything else is just what the machine could actually type.
