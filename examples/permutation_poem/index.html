Permutation Poem

The optional text module rearranges the sentence. The original cut-up engine then passes every permutation through repeated type, a symbolic code column, damaged letter bands, and a modular grid. Meaning and letterform are disturbed in two separate passes.

How to read it: one phrase runs through four passes; follow the numbered order top to bottom.

  • First order (01) leads: it is the source phrase, printed in a heavier pen so the eye lands there first.
  • Left column = the words. They press hard at the top of each order and fade downward.
  • Right column = the same words as code. Each distinct word gets its own symbol (poster legend); the clean top row of every group is the key — decode it against the legend to read the same order as the words on the left. The rows beneath smear into stable dark texture.
  • The band underneath is the phrase dissolving into a decaying letter field.
  • The grid at the bottom is a light-to-dark tonal staircase of the word orders; each cell repeats its 01–06 index from the left margin, so you can compare the orders directly.

Type your own phrase and press Enter to run it through permute() (empty or symbol-only input falls back to I LOVE YOU). New order cycles the strategy — walk (random swaps), rotate (cyclic shifts), lexical (alphabetical), random (seeded shuffle); New seed reshuffles without changing it. The HPGL export plots the code column and grid in a second pen colour. The faint paper grain and centre fold are screen-only and left out of the SVG/HPGL export.

Core snippet
// The whole library, in three lines — this alone is a complete sketch:
const plot = new GysinPlot({ seed: 1960 });
plot.textCutup("I LOVE YOU", 64, 120);   // one line, sliced and reset
plot.draw();

// Everything below is OPTIONAL, layered on top. This example adds, in order:

// 1) the text addon (p5.gysin.text.min.js): permute() rearranges the phrase
//    into orders before any of it is drawn
const regels = GysinText.permute(phrase, {
  seed: posterSeed, limit: GROUPS, order: ORDERS[orderIndex]
}).slice(0, GROUPS);

// 2) each order printed as repeated cut-up rows that press hard, then fade —
//    one decay curve returns the full param set for every repeat
function decay(i, seedTag, overrides, boost = 0) {
  return {
    strokeWeight: Math.max(0.4, 1.3 + boost - i * 0.17), // felt weight decay
    pressure: Math.min(0.8, boost + i * 0.14),           // deeper = more tremor
    slices: 4 + (i % 5), seed: `${posterSeed}:${seedTag}`, ...overrides
  };
}

// 3) a letter band where the phrase dissolves into jittered glyphs
plot.letters(regels.join(""), 58, BAND_Y, 726, 120, { glyphJitter: 0.8 });

// 4) a tonal grid: one hatch-filled cell per order, tightening 14 -> 4, light to dark
plot.rect(x, y, w, h, { fill: "hatch", hatchSpacing: 14 - i * 2 });

plot.draw();
