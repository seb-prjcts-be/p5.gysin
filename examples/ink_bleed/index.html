bleed

Ink Bleed

The additive counterpart. dropout and rubout take line material away; bleed chooses a few contiguous fragments of the written contour and gives only those one or more extra, slightly shifted passes - ink gathering where the pen went back over its own trace. The same sliced phrase is drawn three times; between the rows only the bleed options change.

The ink knob (Ink +/− or the ↑/↓ arrow keys) sets the share of the contour that gets return passes, from 0 single-pass to 0.45 heavily overdrawn. The extra traces carry role: "bleed" and a physical pass number, so SVG (blade) exports the same drawing with every second and later pass stripped - a knife cannot darken a line - while SVG (pen) keeps the build-up.

Core snippet
// The whole library, in three lines - this alone is a complete sketch:
const plot = new GysinPlot({ seed: 1916 });
plot.textCutup("I AM THAT I AM", 46, 170);   // sliced, one clean pass
plot.draw();

// bleed is opt-in, like every disturbance. A share of the contour gets
// extra, slightly shifted passes; the rest stays a single line:
plot.textCutup("I AM THAT I AM", 46, 340, {
  bleed: 0.22,       // share of the written contour length
  bleedPasses: 2,    // max extra passes per chosen fragment (cap 3)
  bleedSpread: 0.8   // how far a return pass drifts off the first
});

// The same drawing, two tools:
plot.downloadSVG("drawing-pen.svg", { tool: "pen" });     // keeps the ink
plot.downloadSVG("drawing-blade.svg", { tool: "blade" }); // single pass only

// stats() measures the return: bleedPaths, overdrawRatio, maxLocalPasses.
