SVG · JSON · HPGL

Plotter Export

A compact export workflow for pen plotters and vector tools. The same generated traces are drawn on canvas and exported as SVG, JSON, or HPGL. Each layer is stroked in its export-pen colour, so the canvas already is the pen map.

How to read it

  • Pen 1 (ink) - the frame and the registration work: a cross in every corner plus a circle and triangle on the diagonal. The frame hesitates mid-edge and the crosses overshoot and fray, the way a hand-set mark runs past its own centre.
  • Pen 2 (red) - the type: the hero word and a typed subtitle (the optional underwood() typewriter, single-stroke at period pitch), which spells the four export layers FRAME / TYPE / FIELD / REG so the plate captions its own pen map.
  • Pen 3 (blue) - the fields: a decaying letter field and an airy full-width band of operator glyphs. No filled masses - the lower band is signs, not tone.
  • Legend - names which pen draws which layer, its path count, and roughly how many seconds it keeps the machine busy (drawn length ÷ draw speed). The status line adds the total paths, millimetres of ink, and estimated plot time.

Controls

  • New variation - or press R - rerolls the plate: a new symbol set and freshly reshaped glyphs, so every variation reads as a different plate.
  • Rougher + / Cleaner − - or the ↑ / ↓ keys - scale every perturbation from a clean plot up toward heavily hand-disturbed. The CLEAN … ROUGH meter shows where the human scale sits between 0 and 2.
  • Reset - returns the plate to its canonical state (variation 0, human x1.00), always one click away.
  • Download SVG / JSON / HPGL - takes the exact plate on screen to a plotter or vector tool. Seed, variation and human scale are printed on the plate, so any export is reproducible.
Core snippet
// The whole library, in three lines - this alone is a complete sketch:
const plot = new GysinPlot({ seed: 7475 });
plot.text("PLOT", 118, 188);   // clean, mechanical text
plot.draw();

// Everything below is OPTIONAL on top. Defaults are all zero, so options are
// opt-in. This example adds, in order:

// 1) per-pen layers: tag each call with a { layer } so one pen == one ink,
//    and the same tags drive the HPGL pen map on export
const penMap = { frame: 1, type: 2, field: 3, registration: 1 };
const ink = { 1: "#151515", 2: "#b5362b", 3: "#244f73" };

// 2) the composition as data: [layer, method, geometry, perturbation]. One pass
//    over the array draws the plate - frame, type, fields, registration -
//    with the stroke derived from the pen map, not written per shape
for (const [layer, method, geom, opts] of shapesFor(composition(variation))) {
  plot[method](...geom, { layer, stroke: ink[penMap[layer]], ...humanize(opts) });
}
plot.draw();

// 3) export the exact plate on screen - same plot, three plotter-ready formats
plot.downloadSVG("plate.svg", { page, optimize: true });
plot.downloadJSON("plate.json", { includeGenerated: true });
plot.downloadHPGL("plate.hpgl", { page, penMap, speed: 20 });

// 4) stats() reports the plate before it plots: paths, ink length, plot time
const stats = plot.stats({ page, drawSpeed: 20, travelSpeed: 60 });
// per pen: stats.layers.type.drawnLength / drawSpeed
// whole plate: stats.estimatedSeconds -> total plot time
