GysinText.permute() · rotate · lexical

Rotations

A permutation does not have to be cut. This sheet turns one phrase two quieter ways: order "rotate" moves the first word to the end, again and again, until the phrase comes back to itself - four words, four turns, one wheel. Order "lexical" walks every arrangement the dictionary allows, in strict sequence. Both are deterministic: the same phrase always turns the same way.

The drawing stays whole on purpose - plain text(), no scissors. The turning itself is the event: the indent of each line is the first word walking to the back of the queue.

Core snippet
// Load the text module, then the phrase turns in one loop:
const plot = new GysinPlot({ seed: 1959 });
GysinText.permute("RUB OUT THE WORD", { order: "rotate" })
  .forEach((row, i) => {
    plot.text(row, 90 + i * 34, 470 + i * 56, { size: 26 });
  });
plot.draw();

// order "lexical" walks every arrangement in dictionary sequence:
GysinText.permute("RUB OUT THE WORD", { order: "lexical", limit: 12 });
The orders never change - they are the mathematics of the phrase. New turn rerolls only the hand that draws them; Download SVG saves the plotter-ready file. R and S do the same from the keyboard.
