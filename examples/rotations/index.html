Rotations

A palimpsest, layered the way the studio layered. The ground is the machine drawing - a roller grid, turned a few degrees off true so it never squares with the frame. Over it, lattice() writes the whole sheet edge to edge in two off-square passes, the hand against the machine. And over everything, the silent script: the five cyclic orders of the phrase (order "rotate" walks the first word to the back), each written large at its own place and its own angle - columns among rows, one upside down. Nothing centres; nothing reads top-left to bottom-right; the sheet is entered anywhere.

Everything stays addressable. The wear knob re-tunes every standing lattice row through update(); freeze the first pass and only the later hand rerolls. The turned export saves the same sheet upside down through page.rotation, so a second plot run wears the pen the other way.

Core snippet
// The whole sheet: write the field, turn the sheet, write across.
const plot = new GysinPlot({ seed: 1960 });
plot.lattice("COME TO FREE THE WORDS", -15, -15, 730, 1190, {
  turns: [4, 94]   // both passes a few degrees off the grid
});

// The silent script over it, at its own places and angles:
GysinText.permute("COME TO FREE THE WORDS", { order: "rotate" })
  .forEach((row, i) => {
    plot.text(row, x[i], y[i], { size: 24, angle: a[i] });
  });
plot.draw();

// The page's own turn: the same sheet exported upside down.
plot.downloadSVG("turned.svg", { page: { width: 700, height: 1160, rotation: 180 } });
The orders never change - they are the mathematics of the phrase. New turn rerolls only the hand; Wear re-tunes every standing lattice row through update(), no rebuild; Freeze first pass makes the first writing the fixed ground the second keeps crossing; SVG turned 180 exports the sheet upside down through page.rotation. Keys: R, N, W, F, S, T.
